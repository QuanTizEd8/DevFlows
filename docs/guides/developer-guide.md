# Developer Guide

Each promoted workflow lives under `workflows/<workflow-id>`:

- `workflow.yaml` is the source reusable workflow.
- `devflow.yaml` describes ownership, release behavior, docs, examples, and
  tests.
- `.github/workflows/<workflow-id>.yaml` is generated by `devflows sync`.

Run the standard checks before opening a pull request:

```bash
task lint
task test
task docs
```

Inherited workflows stay in `workflows/_drafts` until their interface,
documentation, tests, and release policy are reviewed.
